# Privacy Policy

**Radiocove** values your privacy. This application does not collect, store on a central server, or share any personal user data with third parties.

## 1. Data Collection and Storage
Radiocove is a desktop client. All actions performed within the application (favorite stations, application settings, song history, etc.) are stored exclusively on your local device (computer). Neither we nor any third party can access this data.

## 2. Third-Party Services
The application may communicate with the following external services to provide certain features:

- **Radio Stations:** Station search and listing features use the [radio-browser.info](https://www.radio-browser.info) API.
- **Song Recognition:** Uses Shazam services to identify songs.
- **Metadata and Album Art:** Uses iTunes/Apple Music services to fetch artist information and album images.
- **Image Search:** May utilize Google image search results to find station logos.
- **Discord:** If "Discord Rich Presence" is enabled, it uses the Discord API to display the radio station you are listening to on your Discord profile.

The use of these services is subject to the privacy policies of the respective providers. Radiocove does not send any personal data (name, email, etc.) to these services.

## 3. Automatic Updates
The application communicates with update servers via GitHub to check for new versions. This process is for version control purposes only and does not contain personal data.

## 4. Backups
The backup and restore feature is entirely manual. The backup files (.zip) you create are managed and shared only by you.

## 5. Contact
If you have any questions about this policy or the privacy practices of the application, you can reach out by opening an "Issue" on our GitHub project.
